    第14回 モデル検査ツールとその使い方

    この回では，モデル検査ツール NuSMV の導入方法と使い方について説明します．

    はじめに

    NuSMV は，カーネギーメロン大学 (CMU) にて開発された BDD ベースのモデル検査ツール SMV (Symbolic Model Verifier) を再開発し，新たな機能拡張を行った記号モデル検査器である． NuSMV2 には，SMV や NuSMV1 で実装されていた BDD 記号モデル検査機能に加えて，命題充足可能性判定器 (SAT ソルバ) を用いた有界モデル検査技術が統合されている． また，NuSMV2 は OpenSource ライセンスによって配布されており，利用者の目的を問わず，利用，修正，再頒布が認められている．

    NuSMV2 の主な機能は以下の通りである．

    • 同期および非同期の有限状態システムを表現し，計算木論理（CTL）および線形時間論理（LTL）で表現された仕様を，BDDベースおよびSATベースのモデル検査技術で解析することができる．
    • 効率化や，部分的に状態爆発を抑制するためのヒューリスティックが利用可能である．
    • バッチモードに加えて，テキストインターフェースを用いた対話的処理も可能である．

    なお，NuSMV の後継ツールで，同期式の有限および無限状態システムを対象とした解析が可能なツール nuXmv も公開されている．

    NuSMV のインストール

    NuSMV のソースコードや実行可能バイナリ，マニュアル等は公式サイトから入手可能である．

    1. Getting NuSMV からバイナリを入手する．
    2. ダウンロードしたファイル (Windows なら NuSMV-2.6.0-win64.tar.gz) を展開し，/bin/NuSMV あるいは /bin/NuSMV.exe を適当なフォルダへと移動する．

    Mac OS (または Linux) の場合，Homebrew でもインストールできる．

    $ brew install nu-smv

    NuSMV の実行

    NuSMV の実行形式は，

    • インタラクティブモード
    • バッチモード

    の２通りがある．

    インタラクティブモード

    -int オプションをつけて NuSMV を起動するとインタラクティブモードで実行できる．

    $ NuSMV -int
NuSMV >

    インタラクティブモードは対話型シェルを用いた実行形式で，「読み込み〜評価〜出力」を繰り返す． このモードでは，ユーザは NuSMV でモデル検査を行う際の「モデルの読み込み」「BDD の構築」「特性の検証」といったステップを，システムコマンドとして個別に実行する．

    インタラクティブモードでモデル sample.smv を検証する場合のコマンド実行系列の例を以下に示す．

    $ NuSMV -int
NuSMV > read_model -i sample.smv
NuSMV > flatten_hierarchy
NuSMV > encode_variables
NuSMV > build_model
NuSMV > check_ctlspec
-- specification （検査対象となる CTL 式）  is true
NuSMV >

    それぞれ，

    • read_model ： モデルの読み込み
    • flatten_hierarchy ： モジュールやプロセスの展開
    • encode_variables ： 変数の２値エンコード
    • build_model ： BDD モデルの構築
    • check_ctlspec ： CTL 式の検証

    を行うシステムコマンドである．

    バッチモード

    -int オプションをつけずに NuSMV を起動するとバッチモードで実行できる． バッチモードでは，インタラクティブモードでユーザが個別に指定して実行していたシステムコマンドの列を自動的に実行する．

    バッチモードでモデル sample.smv を検証する場合の実行結果を以下に示す．

    $ NuSMV sample.smv
-- specification （検査対象となる CTL 式）  is true

    SMV 言語

    NuSMV のモデルを記述するための専用言語を SMV 言語と呼ぶ． SMV 言語は，完全同期システムから非同期システムまで，また詳細モデルから抽象モデルまでを記述できるよう設計されている． さらに，モジュール化による階層的な記述も可能である． なお，検査対象は有限状態システムに限られるため，データ型は有限である必要がある． SMV 言語で書かれたモデルを SMV プログラムという．

    例題 SMV プログラム

    SMV プログラムの例を以下に示す．

    sample.smv
    MODULE main
VAR
    request: boolean;
    state: {ready, busy};

ASSIGN
    init(state) := ready;
    next(state) := case
        state=ready & request: busy;
        TRUE: {ready, busy};
        esac;

CTLSPEC AG(request -> AF (state = busy))

    SMV プログラムは，変数の定義，遷移関係の記述，そして検査式から構成される． 上述の例では，VAR で指定されたブロックで変数を定義している．

    変数定義部
    VAR
    request: boolean;
    state: {ready, busy};

    この例では，ブール型の変数 request と {ready, busy} のいずれかの値をとる記号型変数 state が定義されている．

    遷移関係は ASSIGN で指定されたブロックで記述されている．

    遷移記述部
    ASSIGN
    init(state) := ready;
    next(state) := case
        state=ready & request: busy;
        TRUE: {ready, busy};
        esac;

    ASSIGN を用いた遷移関係の記述では，それぞれの変数について，初期状態での値と，次状態での値の変化を指定する． 変数の初期状態での値は init を用いて表し，次状態での値は next を用いて表す． また条件分岐に基づく値の代入には case 式を用いる．

    この例では，変数 state の初期状態での値が ready であり，現在の state の値が ready でかつ request が真ならば次の状態での値が busyとなり，そうでなければ非決定的に {ready, busy} のいずれかへと変化するよう指定されている． また，ここでは変数 request の値の変化については指定されていない． このような場合，request の初期状態や次の状態での値は非決定的に選択される．

    CTL 式による検査特性は CTLSPEC で与えられる．

    検査式記述部
    CTLSPEC AG(request -> AF (state = busy))

    ここでは，CTL 式 が検査式として与えられている．

    型について

    NuSMV で利用できる型は，ブール値，整数，列挙型，ワード，そして配列である． それぞれについて簡単に説明する．

    ブール値

    ブール (boolean) 型は記号値 FALSE と TRUE から構成される．

    整数

    整数 (integer) 型は正と負の整数値から構成される． 現在の実装では，整数型の定義域は -231 + 1 〜 231 - 1 となっている．

    列挙型

    列挙 (enumeration) 型は，値の列挙として構成される．

    {stopped, running, waiting, finished}
{2, 4, -2, 0}

    要素の順序に意味は無いが，要素の重複は許されない．

    {stopped, running, waiting}
{running, stopped, waiting} # 上と同じ型
{stopped, stopped, running, waiting} # これはNG

    また，列挙型は記号定数と整数値が入り交じって定義されていてもよいが，ブール値が含まれることは許されない．

    {FAIL, 1, 3, 7, OK}   # これはOK
{FAIL, 1, 3, 7, TURE} # これはNG

    記号定数のみからなるものを symbolic enum 型，記号と整数からなるものを integer-and-symbolic enum 型とそれぞれいう． なお，整数のみからなる列挙型は integer 型と等価である．

    ワード

    符号なしワード (unsigined word) あるいは符号付きワード (signed word) は，ビット値のベクトルを表す型である． 符号付きワードの表す値は 2 の補数に基づく． いずれのワード型もビット幅が定められており，unsigned word[N] 型は 0 〜 2N-1 の値を，signed word[N] 型は -2N-1 〜 2N-1 - 1 の値を表すことができる．

    unsigned word[3] # 3 ビットベクトル
signed word[7]   # 7 ビットベクトル

    配列

    配列は，インデックスの下限と上限，そして要素の型によって定義される．

    array 0..3 of boolean
array 10..20 of {OK, y, z}

    多次元配列も定義できる．

    array 1..8 of array -1..2 of unsigned word[5]

    式について

    定数

    NuSMV では定数として，ブール値，整数値，シンボル，ワード値，範囲指定が利用できる．

    ブール定数は boolean 型で，FALSE か TRUE のいずれかとなる． 整数定数は integer 型で，ある整数値となる． 記号定数は symbolic enum 型で，SMV 言語における識別子 (identifier) として記述される．
